O que são Servers?

Na área de desenvolvimento web, um servidor (server, em inglês) é um computador ou sistema responsável por armazenar, processar e entregar páginas da web aos usuários que as acessam por meio de navegadores.

Quando você cria um site com HTML, CSS e JavaScript, esses arquivos precisam estar em algum lugar acessível para outras pessoas na internet. Esse "lugar" é justamente o servidor.

Como funciona?

  1. O usuário digita o endereço de um site no navegador, como www.exemplo.com.
  2. Esse pedido é enviado a um servidor onde o site está hospedado.
  3. O servidor localiza os arquivos do site (como o index.html) e os envia de volta ao navegador.
  4. O navegador recebe o conteúdo e exibe a página para o usuário.

Tipos de servidores

  • Servidor Web: Responsável por entregar os arquivos da web (HTML, CSS, JS etc.). Exemplos: Apache, Nginx.
  • Servidor de banco de dados: Gerencia dados e responde a consultas feitas por sites ou aplicativos.
  • Servidor de aplicações: Executa a lógica do site (como login, cadastro, etc.), normalmente com linguagens como PHP, Python, Node.js etc.
Resumo

Um servidor é essencial para colocar um site no ar. Ele funciona como um "anfitrião" que envia os arquivos certos sempre que alguém acessa uma página da web.

Como é uma estrutura básica de um HTML?

HTML é uma linguagem de programação marcação.

Um arquivo HTML é composto por uma estrutura padrão que organiza o conteúdo da página. Essa estrutura começa com a declaração <!DOCTYPE html>, que informa ao navegador que o documento segue o padrão HTML5.

Em seguida, temos a tag <html>, que envolve todo o conteúdo da página. Dentro dela, há duas partes principais:

  • <head>: é onde ficam informações invisíveis ao usuário, como o título da aba (<title>), links para arquivos CSS, meta tags e scripts.
  • <body>: contém o que será exibido na tela. Dentro do <body>, normalmente usamos três seções principais:
    • <header>: área superior da página, geralmente com o logotipo, título ou menu de navegação.
    • <main>: parte central do conteúdo, onde ficam os textos, imagens e informações principais da página.
    • <footer>: rodapé da página, que costuma trazer contatos, direitos autorais e links adicionais.

Essa organização ajuda a manter o código limpo, semântico e mais fácil de entender tanto para desenvolvedores quanto para mecanismos de busca.1

1Por exemplo, o uso correto de tags semânticas melhora a indexação do site em motores de busca como o Google.

Acessibilidade na Web

Adicionar acessibilidade ao HTML não é complicado

A acessibilidade é um dos pilares do desenvolvimento web moderno. Criar páginas acessíveis significa garantir que todas as pessoas, inclusive aquelas com deficiências visuais, auditivas, motoras ou cognitivas, possam navegar e compreender o conteúdo do site com facilidade.

O HTML oferece diversos recursos que ajudam a tornar as páginas mais acessíveis. Alguns dos principais são:

Uso de tags semânticas

Tags como (<header>, <main>, <footer>, etc.) ajudam leitores de tela e outros recursos assistivos a entender a estrutura da página e navegar por ela com mais facilidade.

Atributo alt em imagens

O atributo alt (texto alternativo) descreve o conteúdo de uma imagem. É essencial para pessoas que utilizam leitores de tela ou que não conseguem visualizar as imagens.

Uso da tag <label> com formulários

Associar rótulos (<label>) aos campos de formulário ajuda usuários com deficiências visuais a entenderem o que cada campo representa.

Legendas e descrições em vídeos

Além da tag <track> com arquivos .vtt. para legendas, também é importante fornecer audiodescrição e transcrições sempre que possível, ampliando o acesso para pessoas com deficiência visual ou auditiva.

Uso correto de títulos (<h1> a <h6>)

A hierarquia de títulos ajuda a organizar o conteúdo e permite que tecnologias assistivas naveguem facilmente por seções da página.

Entendendo a diferença entre listas ordenadas e não ordenadas

No HTML, as listas são utilizadas para organizar informações de forma clara e estruturada. Existem dois tipos principais: listas ordenadas e listas não ordenadas, e cada uma serve para situações diferentes.

  • Lista ordenada

    A lista ordenada é usada quando a ordem dos itens importa, como em rankings, passo a passo ou instruções. Os itens são numerados automaticamente pelo navegador.

  • Lista não ordenada

    A lista não ordenada é usada quando a ordem dos itens não é relevante, como em listas de características, tópicos ou ingredientes. Os itens são marcados com pontos (bolinhas) por padrão.

Resumo:
  • Use <ol> quando a sequência for importante.
  • Use <ul> para listar itens sem ordem específica.

Ambas utilizam a tag <li> para definir cada item da lista.

Essas listas são simples, mas muito úteis para deixar o conteúdo mais organizado e acessível tanto para usuários quanto para tecnologias assistivas.

Formulários na Web

Os formulários em HTML são fundamentais para a interação entre o usuário e o site. É por meio deles que coletamos dados, como informações de login, cadastros, comentários, entre outros.

A estrutura básica de um formulário começa com a tag <form>, que agrupa todos os campos e define para onde os dados serão enviados.

Principais elementos de um formulário

  • <input>: usado para diferentes tipos de entrada, como texto, e-mail, senha, data, etc.
  • <label>: rótulo descritivo para cada campo, importante para acessibilidade.
  • <textarea>: área para inserir textos longos, como mensagens ou descrições.
  • <button> ou <input type="submit">: usados para enviar o formulário.

Agrupando campos com <fieldset> e <legend>

Para deixar o formulário mais organizado, podemos agrupar campos relacionados usando a tag <fieldset>. Dentro dela, usamos <legend> para dar um título a esse grupo de campos.

O uso de <form>, junto com elementos como <input>, <label>, <fieldset> e <legend>, permite criar formulários bem estruturados, acessíveis e fáceis de usar. Esses recursos tornam a experiência do usuário mais intuitiva e organizada.
